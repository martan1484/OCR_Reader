Table Template – YAML Configuration

The Table template is used to extract structured row-based data from documents such as invoices, product lists, reports, or logs. Configuration is defined in a YAML file edited manually in a text editor.

Requirements

  • Any text editor (VS Code, Notepad++, etc.)
  • Basic understanding of YAML syntax
  • Clean input documents (PDF, image, scan)

YAML Rules

  • Use spaces for indentation (no tabs)
  • Structure is defined by indentation levels
  • Key-value pairs use :
  • Lists use -
Incorrect indentation will break parsing.

Template Structure

template:
  type: table
  name: example_table
  source: pdf

Columns Definition

Each column defines extracted data fields.

columns:
  - name: item
    type: text
  - name: quantity
    type: number
  - name: price
    type: currency
  • text – general string values
  • number – numeric values
  • date – date values
  • currency – monetary values

Row Detection

Row detection defines how OCR separates table rows in the document. 

row_detection:
  mode: dynamic
  tolerance: 5

Full Example

template:
  type: table
  name: invoice_items

source:
  type: pdf

columns:
  - name: item
    type: text
  - name: quantity
    type: number
  - name: price
    type: currency

row_detection:
  mode: dynamic
  tolerance: 5

Best Practices

  • Use high-quality scans for best OCR accuracy
  • Keep column alignment consistent in documents
  • Start with a minimal configuration and extend gradually
  • Validate YAML before using it in OCR Reader
